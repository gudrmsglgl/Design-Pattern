## State Pattern

> 객체가 상태를 직접 구현하지 않고 <br>
> 상와 행위를 구현한 별개의 상태객체에 <br>
> 위임하는 패턴

### 📍 절차
- 상태 인터페이스 추출
  - [x] fun 상태변경 (상태를 가지고 있는 객체)
    > 상태를 가지고 있는 객체를 파라미터로 넘겨줘야 <br> 
      각 상태객체에서 상태 변경 구현시 <br>
      상태객체의 변경이 가능함
   
- 상태 세분화 구현 객체
- 상태를 가지고 있는 객체 구현
    - [x] 상태 변경 시 상태 객체에게 위임하여 처리

### 실행 출력
```
현재 배달 상태: 배차대기 

배달 상태 다음단계로 변경: 배치완료
배달 상태 다음단계로 변경: 픽업완료
배달 상태 다음단계로 변경: 전달완료
배달 상태 이전단계로 변경: 픽업완료 

현재 배달 상태: 픽업완료
```
